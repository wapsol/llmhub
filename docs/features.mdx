LLMHub provides a unified API to interact with multiple LLM providers without managing individual integrations.

#### Provider Features

- **Automatic Provider Selection**: Route requests to the best provider based on cost, speed, or availability
- **Failover Support**: Automatically retry with alternative providers on failures
- **Model Management**: Enable/disable specific models per provider
- **Cost Optimization**: Choose models based on your budget and performance needs

### Real-Time Cost Tracking

<Info>
  Built on **TimescaleDB** for efficient time-series data storage and automatic aggregations.
</Info>

#### What Gets Tracked

- **Per-Request Costs**: Input and output token counts with accurate pricing
- **Provider Breakdown**: See which providers are most cost-effective
- **Client Attribution**: Track usage and costs per API client
- **Time-Series Data**: Hourly, daily, and monthly cost aggregations

#### Continuous Aggregations

```sql
-- Automatically maintained views
SELECT * FROM llm_hourly_costs;   -- Real-time hourly summaries
SELECT * FROM llm_daily_costs;    -- Daily cost totals
SELECT * FROM llm_monthly_costs;  -- Monthly billing reports
```

#### Budget Management

- **Client Budget Limits**: Set monthly spending caps per API client
- **Rate Limiting**: Control requests per minute for each client
- **Usage Alerts**: Monitor when clients approach budget thresholds
- **Cost Analytics**: Visualize spending trends in the web console

  <Accordion title="api_clients - API Key Management">

### Error Handling

Comprehensive error handling with detailed responses.

```json Response Format
{
  "detail": {
    "error": "authentication_failed",
    "message": "Invalid API key provided",
    "code": "AUTH_001",
    "timestamp": "2025-10-28T10:30:00Z"
  }
}
```

#### HTTP Status Codes

- `200 OK`: Successful request
- `400 Bad Request`: Invalid request parameters
- `401 Unauthorized`: Missing or invalid API key
- `403 Forbidden`: Client is inactive or over budget
- `422 Unprocessable Entity`: Validation errors
- `429 Too Many Requests`: Rate limit exceeded
- `500 Internal Server Error`: Server-side errors
